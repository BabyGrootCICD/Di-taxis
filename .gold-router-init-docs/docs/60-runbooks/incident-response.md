# Runbook: Incident Response

## Severity levels
- SEV1:
- SEV2:
- SEV3:

## Immediate actions
1. Contain (disable trading, revoke keys)
2. Preserve evidence (logs, timestamps)
3. Assess blast radius
4. Communicate status

## Investigation
- Timeline reconstruction
- Root cause analysis
- Remediation plan

## Recovery
- Validate integrity
- Restore service
- Monitor for recurrence

## Post-incident
- Write incident report
- Create follow-up tickets and ADRs if decisions change
